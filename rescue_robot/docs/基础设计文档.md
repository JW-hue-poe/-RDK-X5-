# 救援机器人基础设计文档

## 1. 项目概述

### 1.1 项目背景
本项目是一款基于 RDK 平台的塌方废墟救援寻路机器人，采用**纯视觉感知方案**，无需激光雷达、超声波或 IMU 等传感器，仅通过单目摄像头实现环境感知与导航决策。

### 1.2 核心目标
- **人体检测与测距**：实时识别废墟中的被困人员并估算距离
- **深度估计与避障**：基于单目深度模型分析可通行区域
- **自动跟踪**：云台自动对准检测到的人体目标
- **可视化监控**：实时显示检测结果与状态信息

### 1.3 技术栈
| 类别 | 技术选型 |
|------|----------|
| 框架 | ROS2 Humble Hawksbill |
| 硬件平台 | RDK X3 / X5 |
| 相机 | GS130W MIPI / USB 摄像头 |
| 人体检测 | YOLOv8 / YOLO11 (BPU 加速) |
| 深度估计 | Depth Anything V2 (BPU 加速) |
| 云台控制 | PCA9685 + SG90 舵机 |

## 3. 核心组件详解

### 3.1 图像采集节点 (`camera_node`)

**功能定位**：统一图像采集入口，支持 MIPI 和 USB 两种相机模式，发布标准化的图像消息。

**关键特性**：
- **多模式支持**：MIPI 相机代理模式、USB 相机直连模式、仿真测试模式
- **硬件去畸变**：支持 GDC 硬件去畸变（GS130W 推荐开启）
- **自动重连**：USB 相机断开后自动尝试重连
- **图像格式转换**：支持 NV12、BGR8、RGB8 编码转换

**核心代码**：[camera_node.py](file:///home/sunrise/ros2_ws/src/rescue_robot/rescue_robot/camera_node.py)

---

### 3.2 人体检测节点 (`body_detector_node`)

**功能定位**：基于 YOLO 模型实现人体检测，结合单目测距算法估算目标距离。

**关键特性**：
- **BPU 优先**：优先使用 RDK BPU 加速推理，支持 YOLOv8/YOLO11 模型
- **多级回退**：BPU → OpenCV DNN → 测试模式
- **目标跟踪**：基于 IOU 的简单跟踪，分配稳定 ID
- **单目测距**：基于相似三角形原理，考虑相机俯仰角修正

**测距算法原理**：
```python
distance = (fy * average_height * cos(pitch)) / pixel_height - camera_height * sin(pitch)
```

**核心代码**：[body_detector_node.py](file:///home/sunrise/ros2_ws/src/rescue_robot/rescue_robot/body_detector_node.py)

---

### 3.3 深度估计节点 (`depth_estimator_node`)

**功能定位**：基于 Depth Anything V2 模型实现单目深度估计，分析可通行区域。

**关键特性**：
- **深度推理**：支持 BPU 和 ONNX 两种推理方式
- **ROI 分析**：提取车前核心区域进行深度分析
- **可通行掩码**：基于深度阈值生成可通行区域掩码
- **偏移计算**：计算可通行区域中心相对于图像中心的偏移量

**避障策略**：
- 安全距离阈值：0.8m（深度小于此值判定为障碍）
- 危险距离阈值：0.4m（排除地面噪点）
- 形态学处理：开运算去噪 + 闭运算补全

**核心代码**：[depth_estimator_node.py](file:///home/sunrise/ros2_ws/src/rescue_robot/rescue_robot/depth_estimator_node.py)

---

### 3.4 云台控制节点 (`camera_servo_node`)

**功能定位**：控制 SG90 舵机云台，支持手动遥控和自动跟踪两种模式。

**关键特性**：
- **PCA9685 驱动**：I2C PWM 驱动器控制舵机
- **模拟模式**：无硬件时可进行逻辑测试
- **自动跟踪**：根据人体检测结果自动调整云台角度
- **速度限制**：平滑转动，避免机械冲击

**自动跟踪算法**：
```python
# 像素偏移 → 角度增量
offset_x = cx - image_width / 2
angle_per_pixel = hfov_deg / image_width
delta = -offset_x * angle_per_pixel * kp
```

**核心代码**：[camera_servo_node.py](file:///home/sunrise/ros2_ws/src/rescue_robot/rescue_robot/camera_servo_node.py)

## 6. 算法原理

### 6.1 单目人体测距

基于相似三角形原理，通过人体在图像中的像素高度估算距离：

```
           相机光心
              |\
              | \
              |  \ fy (焦距)
              |   \
              |    \
              |_____\
            cy      cx
              |
              | camera_height
              v
        地面

人体实际高度 H = 1.7m
图像中人体像素高度 h_pixel
相机焦距 fy（像素）
俯仰角 θ（向下为正）

距离计算公式：
distance = (fy * H * cosθ) / h_pixel - camera_height * sinθ
```

### 6.2 可通行区域分析

1. **深度图获取**：通过 Depth Anything V2 模型获取单目深度图
2. **ROI 提取**：截取图像底部区域（车前核心区域）
3. **阈值分割**：深度 > 安全距离判定为可通行
4. **形态学处理**：开运算去噪，闭运算补全
5. **偏移计算**：寻找最宽可通行区间，计算中心偏移量

### 6.3 自动云台跟踪

1. **目标选择**：选择面积最大的人体检测框
2. **偏移计算**：计算目标中心与图像中心的像素偏移
3. **角度转换**：像素偏移 × 每像素角度 × 比例增益
4. **平滑控制**：限制最大角速度，避免机械冲击

## 8. 启动与使用

### 8.1 环境准备

```bash
# 安装依赖
pip install opencv-python cv_bridge smbus2

# 复制模型文件到目标目录
mkdir -p /opt/rescue_robot/models
cp your_model.bin /opt/rescue_robot/models/
```

### 8.2 启动 MIPI 相机（GS130W）

```bash
ros2 launch rescue_robot mipi_cam_gs130w.launch.py
```

### 8.3 启动救援机器人视觉节点

```bash
ros2 launch rescue_robot rescue_robot.launch.py
```

### 8.4 单目模式启动

```bash
ros2 launch rescue_robot rescue_robot_monocular.launch.py
```

### 8.5 手动控制云台

```bash
# 向左转 30 度
ros2 topic pub /rescue_robot/cmd_camera_pan std_msgs/msg/Float64 "{data: -30.0}"

# 回中
ros2 topic pub /rescue_robot/cmd_camera_pan std_msgs/msg/Float64 "{data: 0.0}"
```

## 10. 扩展与对接

### 10.1 导航决策层

深度估计节点发布的 `/rescue_robot/passable_offset` 可直接用于导航决策：
- 正数 → 向右转
- 负数 → 向左转
- 接近 0 → 直行

### 10.2 状态机集成

`visualization_node` 订阅 `/rescue_robot/state` 话题，支持机器人状态显示。上层状态机可发布此话题：

```python
from std_msgs.msg import Int32
from rescue_robot.utils import RobotState

pub = self.create_publisher(Int32, '/rescue_robot/state', 10)
pub.publish(Int32(data=int(RobotState.HUMAN_DETECTED)))
```

### 10.3 远程监控

## 附录：参数调整建议

| 参数 | 调整场景 | 建议值 |
|------|----------|--------|
| `conf_threshold` | 检测3D打印小人时 | 降低到 0.2 |
| `average_human_height` | 不同人群场景 | 儿童: 1.2-1.5m, 成人: 1.6-1.8m |
| `camera_height` | 相机安装高度变化 | 测量实际安装高度 |
| `safe_distance` | 不同速度场景 | 低速: 0.5m, 高速: 1.0m |
| `kp` | 跟踪灵敏度 | 迟钝: 0.3-0.5, 灵敏: 0.8-1.0 |
