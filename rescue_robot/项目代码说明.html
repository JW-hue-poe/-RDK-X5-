基于RDK单目视觉的救援机器人 - 视觉-only版本

本代码包仅包含视觉感知部分，提供图像采集、人体检测、单目深度估计、可通行区域分析和可视化节点。决策规划与底盘控制未包含，可订阅本包发布的话题自行接入。

项目根目录：rescue_robot/ 位于当前工作空间下。在 RDK 板上可放入 ~/ros2_ws/src 后执行 colcon build 编译。

二、核心节点

camera_node.py 核心 ▶

负责图像采集与发布。

  • 支持 USB 摄像头（OpenCV）和 MIPI 相机转发两种模式
  • 发布 /rescue_robot/image_raw 和 /rescue_robot/camera_info
body_detector_node.py 核心 ▶

人体检测与单目测距。

  • 优先使用 BPU 模型，未找到则回退到 ONNX / 测试模式
  • 基于人体平均身高和检测框像素高度实现相似三角形测距
  • 带简单 IOU 跟踪，避免重复识别
  • 发布 /rescue_robot/body_detections 和 /rescue_robot/nearest_human_distance
depth_estimator_node.py 核心 ▶

单目深度估计与可通行区域分析。

  • 可加载 Depth Anything V2 Small ONNX 或订阅外部深度节点
  • 对车前 ROI 做深度阈值分割，区分障碍与可通行区域
  • 计算可通行区域中心偏移，引导转向
  • 发布 /rescue_robot/depth_image、/rescue_robot/passable_mask、/rescue_robot/nearest_obstacle_distance、/rescue_robot/passable_offset
visualization_node.py 核心 ▶

可视化与日志。

  • 叠加人体框、距离、状态、深度图小窗
  • 发布 /rescue_robot/overlay_image
  • 定时发布搜救统计日志
actuator_bridge_example.py 核心 ▶

视觉 -> 电机/舵机桥接示例。

  • 订阅视觉节点输出的人体、障碍、可通行区域信息
  • 根据优先级生成 /rescue_robot/cmd_vel 速度指令
  • 提供 twist_to_actuators() 示例函数，可直接替换为实际电机 PWM / 舵机角度
  • 他人的电机/舵机驱动代码只需订阅 /rescue_robot/cmd_vel 或参考此节点改造

三、辅助工具

camera_calibration.py 工具 ▶

棋盘格相机标定脚本，输出可直接替换 config/camera_params.yaml。

python3 scripts/camera_calibration.py --collect --device 0 --square_size 0.025
deploy_models.sh 工具 ▶

RDK 板端模型部署脚本，负责复制 TROS 官方人体检测模型并转换 Depth Anything V2 Small 为 BPU bin。

bash scripts/deploy_models.sh
offline_simulation.py 工具 ▶

无需 RDK 和 ROS2 的离线仿真，在 PC 上验证视觉算法链路。

python3 scripts/offline_simulation.py

四、RDK GS130w 单目 MIPI 摄像头使用说明

本项目默认使用 RDK GS130w 单目 MIPI 摄像头。使用流程如下：

  1. 确认 config/camera_params.yaml 中 camera.camera_type 为 mipi，mipi_topic 指向 /mipi_cam/image_raw。
  2. 安装并启动 TROS 官方 mipi_cam 节点： 
    sudo apt install tros-hobot-mipi-cam
ros2 launch rescue_robot mipi_cam_gs130w.launch.py
  3. 启动视觉节点： 
    ros2 launch rescue_robot rescue_robot.launch.py

单摄像头即可同时用于人体检测和深度估计，两个视觉节点共享同一个 /rescue_robot/image_raw 话题。

五、部署到 RDK

Windows 电脑推荐使用 PowerShell 一键部署脚本：

# 1. 修改 scripts/deploy_to_rdk.ps1 中的 RDK_IP
# 2. 执行部署
cd rescue_robot
.\scripts\deploy_to_rdk.ps1

脚本会自动打包、通过 scp 上传、ssh 解压编译。部署完成后 SSH 登录 RDK 启动即可。

如果你只想手动用 scp 传，详见 docs/SCP手动部署指南.md，核心命令如下：

# 1. 打包
cd C:\Users\jingwai\WorkBuddy\2026-07-05-00-10-44
tar --exclude='.git' --exclude='__pycache__' --exclude='*.pyc' -czf rescue_robot.tar.gz rescue_robot

# 2. SCP 上传到 RDK（替换 IP）
scp -P 22 rescue_robot.tar.gz root@192.168.1.100:/tmp/

# 3. 在 RDK 上解压编译
ssh root@192.168.1.100 "bash -c 'mkdir -p /root/ros2_ws/src && rm -rf /root/ros2_ws/src/rescue_robot && tar -xzf /tmp/rescue_robot.tar.gz -C /root/ros2_ws/src/ && cd /root/ros2_ws && source /opt/tros/setup.bash && colcon build --packages-select rescue_robot --symlink-install'"

详细步骤见 docs/RDK部署指南.md 和 docs/SCP手动部署指南.md。

六、核心话题

  • /rescue_robot/image_raw — 原始相机图像
  • /rescue_robot/body_detections — 人体检测结果
  • /rescue_robot/nearest_human_distance — 最近人体距离
  • /rescue_robot/depth_image — 深度热力图
  • /rescue_robot/passable_mask — 可通行区域掩码
  • /rescue_robot/nearest_obstacle_distance — 最近障碍距离
  • /rescue_robot/passable_offset — 可通行区域中心偏移
  • /rescue_robot/overlay_image — 叠加可视化图像

七、电机/舵机驱动对接

视觉节点只负责感知，不直接控制电机/舵机。推荐以下三种方式：

方案1：订阅 /rescue_robot/cmd_vel ▶

本包已提供 actuator_bridge_example.py，将视觉结果转换为 geometry_msgs/Twist 发布到 /rescue_robot/cmd_vel。他人的电机/舵机驱动节点只需订阅该话题。

ros2 launch rescue_robot rescue_robot_with_bridge.launch.py
方案2：直接订阅视觉话题 ▶

他人的驱动代码也可直接订阅：

  • /rescue_robot/body_detections：人体位置与距离
  • /rescue_robot/nearest_human_distance：最近人体距离
  • /rescue_robot/nearest_obstacle_distance：最近障碍距离
  • /rescue_robot/passable_offset：可通行方向
方案3：修改 actuator_bridge_example.py ▶

如果 cmd_vel 不适合你的底盘，直接修改 twist_to_actuators() 函数，把 Twist 转换为你自己的电机 PWM 或舵机角度。
