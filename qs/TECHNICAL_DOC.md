## 1. 系统概述

### 1.1 应用场景

本系统应用于**地震坍塌场景**下的智能救援机器人，主要功能包括：
- 在复杂废墟环境中自主行走与避障
- 通过摄像头识别被困人员
- 利用LoRa通信模块向救援指挥中心传输现场信息
- 支持多种姿态控制以适应不同地形

### 1.2 整体架构

系统采用**分层架构**设计，共分为三层：

| 层级 | 名称 | 职责 | 包含模块 |
|------|------|------|----------|
| **感知层** | App | 摄像头识别、AI决策、数据处理 | `ds_api.py` |
| **控制层** | Control | 姿态控制、运动协调 | `arm_wheel.py` |
| **驱动层** | Driver | 硬件驱动、底层通信 | `motor.py`, `servo.py`, `pca9685.py`, `lora.py` |

## 2. 模块功能说明

### 2.1 Driver 层

#### 2.1.1 pca9685.py - PWM驱动芯片

**功能描述**：提供PCA9685芯片的底层驱动，实现16通道PWM信号生成。

**核心接口**：

| 接口 | 功能 | 参数 | 返回值 |
|------|------|------|--------|
| `__init__(bus_num, addr)` | 初始化PCA9685 | bus_num: I2C总线号；addr: 设备地址 | PCA9685实例 |
| `set_pwm_freq(freq)` | 设置PWM频率 | freq: 24~1526 Hz | None |
| `set_duty_cycle(ch, duty)` | 设置通道占空比 | ch: 通道号(0~15)；duty: 占空比(0.0~1.0) | None |
| `set_raw_pwm(ch, on, off)` | 设置原始PWM值 | ch: 通道号；on/off: 0~4095 | None |
| `get_channel_raw(ch)` | 读取通道状态 | ch: 通道号 | (on, off) |
| `single_channel_zero(ch)` | 关闭单个通道 | ch: 通道号 | None |
| `all_channel_zero()` | 关闭所有通道 | 无 | None |
| `close()` | 关闭总线连接 | 无 | None |

**数据流向**：
```
上层调用 ──▶ set_duty_cycle() ──▶ set_raw_pwm() ──▶ I2C总线 ──▶ PCA9685寄存器
```

#### 2.1.2 motor.py - 电机驱动

**功能描述**：控制四轮电机的方向和速度，支持前进、后退、左转、右转等基本运动。

**硬件配置**：
- 电机数量：4个（左前、右前、左后、右后）
- 控制方式：通过PCA9685的8个PWM通道控制（每个电机AIN/BIN各占1通道）
- 通道映射：

| 电机 | AIN通道 | BIN通道 |
|------|---------|---------|
| 左后轮 | CH0 | CH1 |
| 右后轮 | CH2 | CH3 |
| 左前轮 | CH4 | CH5 |
| 右前轮 | CH6 | CH7 |

**核心接口**：

| 接口 | 功能 | 参数 | 返回值 |
|------|------|------|--------|
| `__init__(pca_driver)` | 初始化电机控制器 | pca_driver: PCA9685实例 | CarMotor实例 |
| `car_forward()` | 四轮正转（直行） | 无 | None |
| `car_backward()` | 四轮反转（后退） | 无 | None |
| `car_left_group_run()` | 左侧电机运行（原地左转） | 无 | None |
| `car_right_group_run()` | 右侧电机运行（原地右转） | 无 | None |
| `car_stop()` | 停止所有电机 | 无 | None |

**数据流向**：
```
arm_wheel调用 ──▶ car_forward() ──▶ set_duty_cycle() ──▶ pca9685 ──▶ 电机驱动器(TB6612)
```

#### 2.1.3 servo.py - 舵机驱动

**功能描述**：控制关节舵机的角度，支持角度设置、使能/禁用控制。

**硬件配置**：
- 舵机数量：4个（左前关节、右前关节、左后关节、右后关节）
- 控制方式：通过PCA9685的通道8~11控制
- 角度范围：0° ~ 180°
- PWM频率：50Hz

**核心接口**：

| 接口 | 功能 | 参数 | 返回值 |
|------|------|------|--------|
| `__init__(pca_driver)` | 初始化舵机控制器 | pca_driver: PCA9685实例 | ServoController实例 |
| `add_servo(name, pwm_ch)` | 添加舵机 | name: 名称；pwm_ch: PWM通道 | SingleServo实例 |
| `get_servo(name)` | 获取舵机 | name: 名称 | SingleServo实例 |
| `disable_all()` | 禁用所有舵机 | 无 | None |
| `cleanup()` | 清理资源 | 无 | None |

**SingleServo接口**：

| 接口 | 功能 | 参数 | 返回值 |
|------|------|------|--------|
| `set_angle(angle)` | 设置舵机角度 | angle: 0~180 | None |
| `get_angle()` | 获取当前角度 | 无 | 当前角度值 |
| `enable()` | 使能舵机 | 无 | None |
| `disable()` | 禁用舵机 | 无 | None |

**数据流向**：
```
arm_wheel调用 ──▶ set_angle() ──▶ _angle_to_pwm() ──▶ pca9685 ──▶ 舵机
```

#### 2.1.4 lora.py - LoRa通信模块

**功能描述**：通过串口与LoRa模块通信，实现远距离数据传输。

**硬件配置**：
- 通信方式：串口（UART）
- 波特率：9600
- 数据位：8
- 停止位：1
- 校验位：无

**核心接口**：

| 接口 | 功能 | 参数 | 返回值 |
|------|------|------|--------|
| `__init__(port_name)` | 初始化LoRa设备 | port_name: 串口号 | LoraDevice实例 |
| `open()` | 打开串口 | 无 | True/False |
| `send(text)` | 发送数据 | text: 字符串 | True/False |
| `get_recv_data()` | 获取接收数据 | 无 | list[bytes] |
| `close()` | 关闭串口 | 无 | None |

**数据流向**：
```
arm_wheel调用 ──▶ send() ──▶ 串口 ──▶ LoRa模块 ──▶ 无线传输 ──▶ 指挥中心
```

### 2.2 Control 层

#### 2.2.1 arm_wheel.py - 姿态控制

**功能描述**：集成舵机控制和电机控制，实现六种姿态控制方法。

**姿态配置**：

| 姿态名称 | 左前关节 | 右前关节 | 左后关节 | 右后关节 | 电机方向 |
|----------|----------|----------|----------|----------|----------|
| 正身 | 90° | 90° | 90° | 90° | - |
| 附身 | 60° | 60° | 120° | 120° | - |
| 趴下 | 30° | 30° | 150° | 150° | - |

**核心接口**：

| 接口 | 功能 | 参数 | 返回值 |
|------|------|------|--------|
| `__init__(bus_num, addr)` | 初始化姿态控制器 | bus_num: I2C总线号；addr: 设备地址 | BodyControl实例 |
| `upright_forward()` | 正身向前 | transition_time: 过渡时间(秒) | None |
| `upright_backward()` | 正身向后 | transition_time: 过渡时间(秒) | None |
| `lean_forward()` | 附身向前 | transition_time: 过渡时间(秒) | None |
| `lean_backward()` | 附身向后 | transition_time: 过渡时间(秒) | None |
| `lie_down_forward()` | 趴下向前 | transition_time: 过渡时间(秒) | None |
| `lie_down_backward()` | 趴下向后 | transition_time: 过渡时间(秒) | None |
| `stop()` | 停止电机 | 无 | None |
| `reset()` | 重置到正身姿态 | transition_time: 过渡时间(秒) | None |
| `get_current_status()` | 获取当前状态 | 无 | 状态字典 |
| `print_status()` | 打印当前状态 | 无 | None |
| `cleanup()` | 清理资源 | 无 | None |

**状态字典结构**：
```json
{
    "posture": "upright",
    "posture_name": "正身姿态",
    "direction": "forward",
    "is_running": true,
    "joint_angles": {
        "lf_joint": 90,
        "rf_joint": 90,
        "lb_joint": 90,
        "rb_joint": 90
    }
}
```

**姿态过渡算法**：
- 使用线性插值实现平滑过渡
- 过渡步数：max(10, transition_time * 50)
- 每步间隔：transition_time / steps

**数据流向**：
```
ds_api决策 ──▶ upright_forward() ──▶ _set_posture() ──▶ servo.set_angle()
                                    ──▶ _set_motor_direction() ──▶ motor.car_forward()
```

### 2.3 App 层

#### 2.3.1 ds_api.py - DeepSeek AI决策

**功能描述**：调用DeepSeek大模型API，根据路况数据进行智能行走决策。

**核心接口**：

| 接口 | 功能 | 参数 | 返回值 |
|------|------|------|--------|
| `__init__(api_key, base_url, default_sys_prompt)` | 初始化AI客户端 | api_key: API密钥；base_url: 接口地址；default_sys_prompt: 默认提示词 | DeepSeek实例 |
| `set_default_prompt(new_prompt)` | 设置默认提示词 | new_prompt: 新提示词 | None |
| `get_reply(user_text, custom_system_prompt)` | 获取模型回复 | user_text: 用户输入；custom_system_prompt: 自定义提示词 | 模型回复字符串 |

**数据流向**：
```
摄像头数据 ──▶ get_reply() ──▶ DeepSeek API ──▶ 决策结果 ──▶ arm_wheel
```

## 3. 核心业务流程

### 3.1 主流程概述

```
┌─────────────────────────────────────────────────────────────────────┐
│                        机器人主循环流程                              │
├─────────────────────────────────────────────────────────────────────┤
│  1. 初始化阶段                                                       │
│     ├─ 初始化PCA9685                                                │
│     ├─ 初始化舵机控制器 (ServoController)                            │
│     ├─ 初始化电机控制器 (CarMotor)                                   │
│     ├─ 初始化姿态控制器 (BodyControl)                                │
│     ├─ 初始化AI决策模块 (DeepSeek)                                   │
│     └─ 初始化LoRa通信模块 (LoraDevice)                               │
├─────────────────────────────────────────────────────────────────────┤
│  2. 感知阶段                                                         │
│     ├─ 摄像头采集现场图像                                            │
│     ├─ 识别世界坐标系与当前路况数据                                   │
│     └─ 提取路况特征（障碍物、坡度、通道宽度等）                       │
├─────────────────────────────────────────────────────────────────────┤
│  3. 决策阶段                                                         │
│     ├─ 构建决策输入（路况特征描述）                                   │
│     ├─ 调用DeepSeek API进行智能决策                                  │
│     └─ 解析决策结果（动作指令：前进/后退/左转/右转/停止）             │
├─────────────────────────────────────────────────────────────────────┤
│  4. 执行阶段                                                         │
│     ├─ 根据决策结果选择姿态                                          │
│     │   ├─ 平坦地形 → 正身姿态                                       │
│     │   ├─ 低矮通道 → 趴下姿态                                       │
│     │   └─ 斜坡/不稳定地形 → 附身姿态                                │
│     ├─ 调用arm_wheel相应方法执行运动                                 │
│     └─ 实时调整电机速度和方向                                        │
├─────────────────────────────────────────────────────────────────────┤
│  5. 通信阶段                                                         │
│     ├─ 收集当前状态信息（姿态、位置、速度）                           │
│     ├─ 收集发现的受难人员信息                                        │
│     └─ 通过LoRa发送至指挥中心                                        │
├─────────────────────────────────────────────────────────────────────┤
│  6. 循环                                                             │
│     └─ 返回步骤2继续执行                                            │
└─────────────────────────────────────────────────────────────────────┘
```
